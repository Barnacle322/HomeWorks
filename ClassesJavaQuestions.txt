1. Что такое конструктор?
Метод который вызывается при иниицилизации объекта. Конструктор позволяет создавать объекты уже с параметрами внутри них, не используя геттеры

2. Что такое дефолтный конструктор (по умолчанию)?
При иницилизации объекта создается дефолтный конструктор "за кулисами", он сам по себе пустой. 
В нем прописаны некоторые значения для разных типов данных типа: null, 0... 
Также, если существует недефолтный конструктор уже существует, то создать дефолтный конструктор можно таким же образом, просто не заполняя никаких аттрибутов

3. Как работает конструктор?
При создании экземпляра класс можно передать аттрибуты этого объекта в скобках. Таким образом данный объект уже будет в себе иметь все эти параметры в рабочем состоянии.

4. Для чего нужен конструктор?
Скорее всего для экономии времени. Место того, чтобы прописывать множество сеттеров, можно передать всю ифнормацию в конструктор.

5. Какой конструктор вызывается когда конструкторов несколько?
Будет вызван тот, который подходит по количеству атрибутов и если типы входных данных совпадают. 
Если существуют два похожих конструктора(одно кол-во аттрибутов и совпадают типы данных), то будет использован тот, который находится выше в коде

6. Что такое this
Если название локальной переменной в сеттере или в конструкторе совпадает с названием переменной одного их параметров класса, то можно использовать this. перед переменной параметра класса, чтобы ссылаться именно на него.
