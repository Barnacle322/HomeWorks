1. Что такое модификаторы доступа и зачем они нужны?
2. Какие бывают модификаторы доступа?

Модификаторы доступа используются для "скрытия" и "защиты" частей кода(аттрибуты, методы, конструкторы и сами классы). Есть 4 вида модификаторов, каждый из которых является более "суровой версией". Public - код доступен для любых классов. Protected - код доступен внутри пакета классов и для подклассов. Priavte - код доступен только внутри класса, в котором он написан. Default - модификатор по умолчанию, код доступен только внутри одного пакета.

3. Что такое package, для чего они нужны?

"Пакеты" нужны для группировки схожих классов. Пакеты являются отдельной директорией, таким образом можно иметь классы с одинаковыми названиями в разных пакетах. Также пакеты - это отличный способ сделать свой проект проще для понимания и работы с ним.

4. В чем отличие между default и public?

При public код будет доступен для всех классов внутри проекта. При default только внутри пакета класса.

5. Зачем делать сеттер private а геттер public, что это нам даст?
Это даст возможность только считывать данные, но не записывать или менять их.
