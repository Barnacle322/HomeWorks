1. Что такое инкапсуляция, приведите инструменты для ее реализации. 
Способ структурирования кода, при котором к его частам(методам, аттрибутам) не будет прямого доступа. Для ограничения доступа используются модификаторы доступа. Public, protected, private, default. Каждый из них имеет свой "уровень защиты".
Но, допустим, есть аттрибут который имеет модификатор private. К нему мы будем обращаться с помощью геттеров и/или сеттеров. Таким образом у нас не будет прямого доступа к переменной, но мы все же ссможем использовать значения из этой переменной.

2. Что такое наследование, приведите 2 примера наследования.
Наследование - это механизм, при котором объект(класс) приобретет все аттрибуты и методы другого класса, родительского. 
Класс родитель - гитара
Классы дети: басс гитара, укулеле, акустика, электрическая гитара...

Класс родитель - велосипед
Классы дети - обычный велосипед, электровелосипед, моноцикл...

3. Что такое static?
Модификатор для аттрибутов и/или методов, который подразумевает, что значения этого аттрибута или метода будет одинаково для всех объектов класса.

4. Для чего мы используем static?
Статик методы можно использовать без инициализации объекта, обращаясь напрямую к классу(Class.getQuantity). Так, например, можно вывести информацию о всех экземплярах класса, к примеру, кол-во этих экземпляров.

5. Можем ли мы унаследовать статичный метод?
Да, можно, но нельзя будет Override его.

6. Как получить значение статичной переменной?
Обратиться напрямую(Class.static) или же через геттер(Class.getStatic()), если переменная имеет модификатор private

7. Как запретить изменение, но разрешить считывание статичной переменной?
Сделать переменную private, брать значение через геттер.
