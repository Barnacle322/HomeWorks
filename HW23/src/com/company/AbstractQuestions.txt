1. Модификатор abstract - non-access модификатор, используемый для достижения абстракции. Методы с этим модификтором не будут иметь в себе никакого кода. Весь рабочий алгортим будет присутствовать в наследниках 

2. К классами и методам

3. Создать abstract метод в классе, унаследовать этот класс, в наследнике уже прописываем всю логику созданного метода через override

4. К примеру надо показать свою программу инвесторам, но есть опасения, что кто-то может просто взглянуть на код и своровать реализацию. С помощью абстракции можно будет скрыть реализацию программы, тем самым защитив себя.

5. Abstract в классах, например, класс родитель abstract, в таком случае он будет всего лишь, как чертеж, без деталей, без тела. Это самое тело, уже дописывается в наследнике. Абстрактные классы нельзя использовать для создания объектов, обратиться к ним можно только при помощи наследников. Такие классы в основном и используют просто для создания базы для классов наследников.
