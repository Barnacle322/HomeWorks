1. Что такое интерфейс?
Интерфейс - это полностью абстрактный класс, используемый для группировки относящихся к друг другу методов, у которых нет тела.

2. В чем преимущество использования интерфейсов?
Интерфейсы позволяют достичь множественного наследования.

7. В чем отличие между наследованием и композицией?
Композиция - это отношение между классами, как "имеет в себе". То есть объекты которые находятся в композиции не могут существовать без других объектов этой самой композиции. К примеру есть класс Книга, у книги есть информация об авторе, и дате выпуска и.т.п. Также есть класс Библиотека в котором есть референс на список с каким-то кол-вом книг. Без библиотека не будет возможности обратиться к этому списку книг. Такое взаимоотношение классов называется композицией.

При наследовании же отношение между классами будет иметь вид "оно является". При наследовании все методы и аттрибуты копируются в дочерний класс из родительского. К примеру есть родительский класс "Гитара", из которого наследуется несколько других дочерних типа "Акустическая", "Электрическая", "Басс" и.т.п. В родительском классе будут прописаны общие "черты" всех дочерних классов, чтобы не надо было их прописывать заново несколько раз. В дочерних классах также можно уже добавить свои методы и аттрибуты пресущие только этому самому классу.
